# Dependency Security Audit — 2026-06-11

## Summary

`npm audit` reports **2 moderate vulnerabilities** after upgrading Next.js from `16.2.6` to `16.2.9`.

The remaining advisory is upstream/transitive:

- `postcss < 8.5.10`
- bundled under `next/node_modules/postcss@8.4.31`
- reported through `next@16.2.9`

npm's suggested automatic fix is to install `next@9.3.3`, which is a major downgrade and not a safe or relevant fix for this Next 16 application. That recommendation was rejected. Civilization survives another day.

## Actions Taken

- Upgraded:
  - `next`: `16.2.6` → `16.2.9`
  - `eslint-config-next`: `16.2.6` → `16.2.9`
- Re-ran:
  - `npm ci --ignore-scripts`
  - `npm run lint`
  - `npm run typecheck`
  - `npm run build`
  - smoke test against production server

## Current Decision

Track the advisory as **upstream pending** until Next releases a stable version that updates its bundled PostCSS dependency without downgrading the app.

Do **not** run `npm audit fix --force` for this issue.

## Follow-up

Periodically check:

```bash
npm view next version
npm audit
```

When a newer stable Next release resolves the bundled `postcss` advisory, upgrade Next and `eslint-config-next` together, then rerun the full CI gate.
