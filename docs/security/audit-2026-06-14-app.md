## Findings

### 1. Dead upstream telemetry in middleware logs client IP — should remove

`src/middleware.ts` fires two `fetch()` calls on **every** non-API page request
to a hardcoded host `http://umami-umami-1:3000/api/send`, including a "Network
Log" event carrying the client IP, with a hardcoded fallback `UMAMI_WEBSITE_ID`.

- This is leftover OSIRIS upstream telemetry. No `umami` container exists on
  beacon-core, so the host never resolves — every page view makes two failed
  fetches (swallowed by `.catch()`, so non-fatal but wasteful).
- For a locally-owned, defensive tool, silently shipping client IPs to a
  third-party analytics endpoint is the wrong default.

**Recommendation:** delete the umami calls (or gate them behind an explicit,
self-hosted, opt-in `UMAMI_HOST` env var with no IP logging).

### 2. Health version string mismatch (cosmetic)

`/api/health` reports `version: "1.0.0"` while `package.json` is `0.1.0`. Pick
one source of truth (read version from `package.json`, or bump the package).
